What Is Sass?

Sass is a scripting language for style sheets. Cascading Style Sheets (CSS) allow a developer to declare the styles, colors, and layouts of a web app, separating it from the content of the app to create better organization and management. Using a series of selectors, CSS groups rules to the various elements on the page.

However, the more elaborate the site becomes the longer and more complex the CSS will be. If you have multiple developers working on the CSS, the task of modification becomes daunting. Sass was designed to change all that. It is a CSS preprocessor, meaning that it will convert your scripts to CSS that can then be compiled. Sass's emphasis is on organizatizing the stylesheets and minimalizing code, leading to efficient management and minimal errors.

Why Sass?

It's simple - You can get more done with less code in less time and your code is more readable and managable. Who doesn't like that?
Anyone can learn - With extensive documentation and an ever-growing community of fans, it's easy for the novice developer to install, implement, and benefit from Sass.
Anyone can use it - Sass was initially written for Ruby, but has now been expanded to include implementations in C, Go, Java, Javascript, Lua, .Net, Node, Perl, PHP, Python, and Scala. No matter what framework and language you use, Sass is there for you.
It's powerful - CSS is limited in the rules that can be applied. Sass is to CSS what object-oriented languages is to programming. With power and elegance, Sass transforms your CSS.

Sassy Syntax

Sass actually has two syntaxes. The first, Indented Syntax (Sass) is the original syntax. It uses indentation to separate code blocks unlike CSS which uses brackets. It also uses newline characters to separate rules as opposed to semi-colons. This made the code more readable than CSS, but for some developers who were used to the brackets and semi-colons of CSS, the fact they had to use a new sytnax for stylesheets frustrated them and prevented them from using some of the hacks they'd developed in CSS.

Sass listened to their followers and made a new sytnax for those loyal CSS fans, Sassy CSS (SCSS). With SCSS, a developer uses brackets to separate code blocks like CSS and semicolons to separate rules. CSS files can be SCSS files, meaning the CSS hacks work again! It also means that vendor-specific syntax (here's looking at you Internet Explorer) also work.

CSS: 
body {
    background-color: blue;
    font-size: 14px;
}
Sass: 
body
    background-color: blue
    font-size: 14px
SCSS: 
body {
    background-color: blue;
    font-size: 14px;
}

Traditional Sass files have the extension .sass, while the new (and more popular) SCSS files have the extension .SCSS. For the remainder of the guide we will use SCSS.
