Getting Started

As mentioned before, Sass was originally implemented in and designed for Ruby. Sass documentation provides concise, easy installation instructions here for Ruby.

In order to use Sass with other languages, the library LibSass was created. With the LibSass library, a wrapper can be utilized to implement it to a language of your choice.

This guide will be using the Javascript implementation: Sass.js

Sass.js can be loaded in a browser using the Sass API or it can be run with Node-Sass for faster performance.

Installing Sass in Node

Before we begin to discuss installing Sass in Node, let’s first review some concepts and elements you’ll need to install Sass and many other packages to create powerful Javascript programs. Package managers have been created to eliminate the need of manual installs and updates of many packages/libraries. Sass is no exception. In order to properly install Sass on your operating system, we'll use Node Package Manager.

The package manager allows you to install Sass from the command line utility and also performs version and dependencies management, assuring that your installation of Sass will work smoothly. Other common Node.js packages requiring package managers are Browserify, Grunt, Bower, Gulp, and Express.

This guide assumes that Node.JS has already been installed on your Operating System. If you have not already installed Node, please do so now

We will run a simple install of Node-Sass on a sample project for demonstration. Feel free to follow along. All commands will be run in command line.

Let's Begin:

In command-line we'll make a sample project

Start by making a directory for the project and change to that directory

mkdir sample && cd sample

Then you can create three directories for the code

mkdir stylesheets stylesheets/scss stylesheets/css

Let's make a SCSS and HTML file:

Notice in the index.html file that we're using a css file as our stylesheet. We haven't made this yet, but node-sass will.

Let's install Node-Sass:

You can either install node-sass in your project locally following the instructions below or install node-sass globally adding -g at the end of the install command.

If you're using Linux/OSX:

npm install node-sass

If you're using Windows:

npm install node-sass –msvs_version=2013

Visual Studio 2013 WD is necessary for compiling versions of Sass 0.9.4 and above on Windows. This flag will ensure the npm manager uses Visual Studio 2013 if you have multiple versions of VS.

If you type 'ls' in command line, you should now see a directory named node_modules which contains everything you need to run the node-sass.

Let's Compile Node-Sass:

To compile, run the following command

./node_modules/.bin/node-sass -o stylesheets/css stylesheets/scss –w

What does this mean? To run node-sass use the script node-sass in the bin directory of node_modules. -o stands for output file and stylesheets/css is the path to the output file(s). stylesheets/scss is the path to the input file(s) that will be translated. The -w at the end of the statement is for watch, allowing the command to continue looking for changes while node-sass is open. To close node-sass enter ctrl + c.

NOTE: This command-line window will need to remain open for the node-sass to run, so open a new command-line or access your files via the sample directory.

Time to write some styling. In your empty main.scss file write the following

main.scss

h1 {
    color: blue;
}

After saving the main.scss file, you should see a message in your command line stating "Rendering Complete, saving .css file...". If you look in the css directory you will now see a file main.css with our scss translated.

Now you're ready to see what Sass can really do!!
