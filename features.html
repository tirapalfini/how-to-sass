Superpowers of Sass

  • Preprocessing - translates your scripts to CSS automatically
  • Variables - store your information and reuse it throughout the stylesheet.
  • Nesting - sass's hierarchy keeps it simple.
  • Partials and Imports- separate your code into snippets that can be included when needed.
  • Mixins - group properties of SCSS and reuse them hroughout the stylesheet
  • Inheritance - 'extend' groups of properties to your selectors
  • Operators - perform mathematical expressions on your properties
  • Functions - dynamically change and create relations between your properties

So without further ado let's dive into these features. There will be examples you can follow along with either in the Sass Playground or with your sample project you created with Node-Sass

Preprocessing

As we've mentioned, CSS does have ways to modularize the stylesheets, separating them into smaller files. However, without a preprocessing language like Sass the developer is left with having to make HTTP requests every time they have to compile the stylesheets at runtime. This slows down performance. Sass is different. Sass compiles all the stylesheets in a single HTTP request, making it efficient and organized. The best of both worlds. Don't worry, we'll see an example of preprocessing when we discuss Imports which is how Sass combines separate files.

Variables

As programmers we have been told to look for redundant code and clean it up. So imagine the frustration of developers who had to deal with writing a styling attribute every time they wrote it. Image worse that the next developer wanted to change it. Not a pleasant or easy task finding and modifying every instance of the attribute. So variables is a big draw for developers to use Sass. Variables can be numbers, strings, colors, lists, maps, and null.

How do you declare a variable?

Variables are declared at the top of the SCSS file (but after the Import files...more on that later). The variable is declared with the $ symbol, similar to Javascript, Java, and PHP. A variable can be scoped globally and locally in the selector

Here's an example

$mainColor: blue;
body {
    background:$mainColor;
}
h1 {
    $mainColor: green;
    color: $mainColor;
}

As you can see, $mainColor is globally declared as blue. A local variable is then declared with the same name in the h1 selector to be green. This is just the beginning of how Sass makes CSS more like a programming language instead of a style sheet language.

Partials and Imports

Besides redundant code, another major pet-peeve of most developers is having a never-ending file of code as opposed to separate, concise files. Sass allows these little concise files, called Partials. These allow for speedy editing and great organization. They are distinguished with an underscore in the beginning of their file name. For example, _footer.scss. To get these snippets of code, the import directive is used. Import your various files into the main file using the @import directive. Any SCSS file can be imported into each other. These imports must be made at the top of the SCSS file though before the variables. CSS allows multiple files, but they combine at runtime each with an HTTP request. Meanwhile, Sass creates a main css file it presents to the web browser. So now you can have many files for components of the web app and they will not interfer with performance. Perfect!

Those who've created a sample project can follow along with the following example: Create the _headers.scss partial, _tables.scss, and modify main.scss in your scss directory

_headers.scss

$mainColor: blue;
$removeMargin: 0;

h1 {
    color: $mainColor;
    text-align: center;
}

h2 {
    margin: $removeMargin;
    $mainColor: green;
    color: $mainColor;
}

_tables.scss

$borderStyle: solid 1px $mainColor;
$fontSize: 20px;

td, th {
    border: $borderStyle;
    font-size: $fontSize;
}

th {
    color: $mainColor;
}

table {
    border-collapse: collapse;
}

Notice that in the _tables.scss file, $mainColor is used but not defined. Running this would give an error message if it was not a partial. However, as a partial it will never be a css file by itself so it's ok. With that being said, the order in which the files are imported matters because variables must be defined before they are used.

main.scss

$space: 15px;

div{
    margin: $space;
}

p, li {
    margin: $removeMargin;
    font-size: $fontSize;
}

Now that the three files are created and saved, it's time to add our import directives. Because we defined $mainColor in the _headers.scss partial, that partial should be imported first. Add 

@import "_headers";
@import "_tables";

to the top of the main.scss file before the variables and save. Notice you don't need the extensions scss, just the name of the file and Sass does the rest. However, if you have css and scss files with the same name you'll need the extension. Also, if your files are in another directory though you will have to include the path.

A main.css file should be created. Open it to take a look. (If we didn't see the friendly 'Rendering complete' in your command-line after creating the files, you may need to 'Control+C' and restart node-sass with the command './node_modules/.bin/node-sass -o stylesheets/css stylesheets/scss –w')

Mixins

Before beginning our discussion on Mixins, let's first discuss CSS vendor prefixes for anyone learning HTML/CSS who is unfamiliar with this concept. As you begin using more and more advanced CSS, you may start to find some browsers support these newer CSS properties, while others don't. Before the browser fully supports the CSS, you can 'experiment' with the CSS features by adding prefixes on. The prefixes allow the browser to try to use your code, but if it isn't supported it won't break the code. With that said, here is an example of a CSS Vendor Prefix for making your borders round: 

-webkit-border-radius: 4px;
-moz-border-radius: 4px;
-ms-border-radius: 4px;
 border-radius: 4px;

To avoid having to write the prefixes over and over for various selectors, you can use a mixin to group the CSS declaration and use it throughout your styles. Take for example you want some of your divs to rotate. To do this you can use the transform CSS property with vendor prefixes. In your main.scss, after import you can copy/paste the folllowing code: 

@mixin rotate($rotation) {
-webkit-transform: rotate($rotation); 
-ms-transform: rotate($rotation);
 transform: rotate($rotation);  
}

You can then use and reuse the mixin anywhere you want in the SCSS file. To do so you simply use the @include directive. For example, in the sample project you can cause the divs for the unordered list and the paragraph to rotate 10 degrees with the following lines:

#skewRight {
    @include rotate(10deg);
}

#skewLeft{
  @include rotate(-10deg);
}

If you are not following along with the sample project and you can see the CSS results in the Sass Playground.

Inheritance

You can have your selectors inherit properties with the @extend directive. This is similar to a mixin, allowing you to pass a properties to the selector. However, unlike a mixin where you are choosing a group of properties to include, with @extend you choose the selector having the properties you want to include on the selector. For this example let's return to our _tables.scss and include a few more properties. You can follow along with your sample project or click the link to go play in the Sass Playground. Erase the previous code in the _tables.scss and copy the following:

$borderStyle: solid 1px $mainColor;
$fontSize: 20px;

table {
  border-collapse: collapse;
}

.header {
    text-align: center;
    padding: 5px;
    color:  $mainColor;
    border: $borderStyle;
    font-size: $fontSize;
}

So we have the header class selector of our table with those properties. Now copy the following code into your _table.scss after the header selector block and complete the code using inheritance:

// Using @extend we can have the .first selector inherit the properties from header.

.first {
    @extend .header;
    color: #333333;
    text-align: left;
}

// Now try to have the second selector inherit all the properties from header except the color, which will be #666666

.second {

}

// Again, have the third selector inherit all the properties from header except color which will be #999999 and text-align which will be right. 

.third {

}

As you can see, it's not copying the properties outright or you would have multiple properties for color and text-align. It's copying the remaining properties from header though, allowing us more concise, easy to follow code.

Operators

Unlike in CSS, Sass actually allows mathematical expressions for our scripting. Operators such as

 + - / * % == != are used to allow more dynamic code. A few things to keep in mind though:

1. Be careful with your units. You may think when you multiply 5px * 10px you're getting 50px, but in reality you now have 50px*px, which CSS doesn't recognize. Instead, multiple 5px * 10 to get 50px.
2. The value units cannot be mixed. Just like some other programming languages, you'll need to convert your units to the like types before performing operations on them. For example, 50px * 10% will give you 500%px, which isn't recognized by CSS.
3. Division must be wrapped in parentheses because in CSS the / symbol is used for the font property such as font: 14px/16px. Therefore, your divion statement could look like $font: (60px/15px) or $font:(50px/5);

Functions

Saving the best for last, functions have allowed Sass to bring CSS to life. Gone are the days of fiddling with the hexidecimal color wheel to find that perfect shade. Instead, leave it to color functions. Numbers can be cast, rounded, and even randomized. Lists and Maps and Selectors oh my! Functions even allow you to check if a variable can exists in your Sass document. For a complete list of functions, check out the Sass Documentation.

And now for a few examples.

Built-In Function

Lets try using the lighten function in leu of choosing hexidecimal colors for the cells of our table Using the previous code in _tables.scss and the function lighten($color, $amount), we can achieve a better fade of the color. 

// Using @extend we can have the .first selector inherit the properties from header.
$cellColor: #000000;
$borderStyle: solid 1px $mainColor;
$fontSize: 20px;

table {
  border-collapse: collapse;
}

.header {
    text-align: center;
    padding: 5px;
    color:  $mainColor;
    border: $borderStyle;
    font-size: $fontSize;
}

//First will have its color 15% lighter than black (#000000)
.first {
    @extend .header;
    text-align: left;
    color: lighten($cellColor, 15%);
}

// Second will have its color 30% lighter
.second {
    @extend .header;
    color: lighten($cellColor, 30%);
}

// Third will have its color 45% lighter  

.third {
    @extend .header;
    color: lighten($cellColor, 45%);
    text-align: right;
}

And because our sample project has such a bold shade of blue, let's soften it a bit by changing the first line of our _headers.scss file to this

$mainColor: lighten(blue, 20%);

Custom Functions

If Sass's pre-built functions just aren't doing the trick, you can just make your own. Below is an example of a custom function in Sass. 

@function calculate-size($size, $factor) {
    @return $size * $factor;
}

To watch this work, copy it in your _headers.scss file and add the following line to the h1 and h2 selectors.

 
// In h1
  font-size: calculate-size(30px, 2);

// In h2
  font-size: calculate-size(20px, 2);
